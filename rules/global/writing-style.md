# Writing style (instruction files)

How to write durable instruction files: rules, CLAUDE.md / agents.md, skills, commands, decisions,
handoffs. (Owner-facing replies: working-style.md. Identifier + filename spelling: naming-discipline.md.
One-fact-one-home + reference-not-restate: anti-stale.md.)

- Terse imperative. Lead with the point, then the why. Cut preamble, fluff, hedging.
- No em dashes. Use a colon or split the sentence.
- Reference, do not restate. Link the canonical source; never paraphrase it.
- One screen where you can. If a file needs scrolling to grasp, it is doing too much: split it or
  move detail behind a pointer (a skill, a path-scoped rule, an index).
- Concrete over abstract. A path, a command, or an example beats a description of one.
- Spell names out. No unexplained abbreviations or acronyms.
- Every line earns its place. Delete one before adding one. A line that repeats another is drift.
