# Knowledge capture

- Personal debugging notes, preferences, and temporary context go to memory, not project files.
- Do not duplicate knowledge. If the current task already produces the relevant document,
  comment, or example, capture it there once; do not restate it elsewhere.
- Before creating a new top-level document, ask first. This is the one deliberate exception
  to "act by default": a stray top-level doc is cheap to create and expensive to find later.
- General lessons go to the global lane; project-specific facts to the project lane. A lesson
  with both a general concept and a specific instance is written in both, linked.
