## Gotchas for the next session

- The plugin cache is keyed by `version` in `plugin.json`. Bumping `0.1.0` will create a new install
  path; old `installPath` remains until uninstalled. Use `claude plugin update claude-kit-core` after
  releasing a new version, then re-verify hooks resolve under the new path.
- Workspace-trust dialog: `claude -p` SKIPS it (documented), so headless smoke tests run hooks. In
  interactive Code sessions, hooks + the memory lane stay nullified until trust is accepted (handoff
  009 / decision 0005). Already accepted in this repo.
- `autoMemoryDirectory` MUST stay at user scope. Removing it there would silently disable the
  memory lane in unrelated repos. Both project and user settings currently set it to the same
  `~/code/claude-kit/memory` path (intentional: the project value is the explicit local override).
- `claude plugin install` does not have an `--enable=false` flag; new installs land enabled. To
  install without enabling, follow with `claude plugin disable`. Not needed here.
